# Contributing

Thank you for helping improve DS Style Customiser.

## Before Submitting Changes

- Keep changes focused and compatible with both supported cartridge models where applicable.
- Do not commit generated executables, project folders, kernel source trees, templates, or build output.
- Test the application from source.
- If your change affects generated kernel configuration, test a build for both cartridge models.

## Reporting Bugs

Include:

- The Customiser version
- Cartridge model
- Steps to reproduce the issue
- Any displayed error message
- Whether the issue occurs in a fresh project

## Pull Requests

Describe what changed, why it was needed, and how it was tested.
