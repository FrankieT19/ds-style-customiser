[INTRO]

Thank you for downloading DS Style Customiser v2.0.

For DS Style v7.2.

DS Style Customiser lets you create a personalised DS Style kernel while keeping the original source untouched.

#####################################################################################

[REQUIREMENTS]

DS Style Customiser needs devkitPro installed at:
C:\devkitPro

Download devkitPro from:
https://github.com/devkitPro/installer/releases

GIMP is recommended for editing the included templates.

Download GIMP from:
https://www.gimp.org/downloads/

#####################################################################################

[BASIC INSTRUCTIONS]

1. Open the provided templates in GIMP.

2. Place your background image on the layer below the UI elements.

3. Edit the UI elements if you want to, but keep them in the same positions.

4. Export your finished images as PNG, JPG, or BMP files.

5. Open DS Style Customiser, create a project, and load your images.

6. When you are finished, use the Build page to create your kernel file.

7. Put the new kernel file on the root of your SD card and hold R as the cartridge boots.

#####################################################################################

[TOP BAR]

The Images page offers two ways to handle the top 19 pixels of each screen:

Separate image keeps the standard DS Style layout and places the selected top bar over each background.

Use screen background displays the upper 19 pixels of each 240 x 160 background instead. Separate top bar images are ignored, allowing each screen to use one continuous full-screen design.

#####################################################################################

[PROJECTS]

Your edits are saved into project folders so you can return to them later.

The customiser keeps the original DS Style source folders untouched.

Choose the correct cartridge model when creating a project. The Omega Definitive Edition uses ezkernelnew.bin, while the original Omega uses ezkernel.bin.

Use Import / Convert to bring an older project onto the current DS Style source, or to create a copy for the other cartridge model. The converted project keeps the original project untouched.

When converting to the original Omega, separate Custom Theme and Custom Colour assets are preserved in the project but are not included in the original Omega build. Edit one of the built-in colours and its top bar and icons instead.

#####################################################################################

[NOTES]

The Custom columns add new Theme and Colour options to Omega Definitive Edition builds. The original Omega has a smaller kernel limit, so customise one of its built-in colour slots instead.

Sound files need to be WAV format. Trim silent sections from the beginning and end for best results.

Use the recommended image sizes shown inside the customiser for best results.

The included templates can be found in the Templates folder.
